Week 12 Lab: Deploying React and Node Apps using Render

Objective

In this lab, you will modify the provided code to use environment variables in both the React and Node.js servers. You will deploy the project to GitHub, create a Render account, connect Render to the appropriate repositories, and set up the environment variables in Render. By the end, your deployed application should correctly display a smiley face emoji when everything is set up properly, and a frown emoji if something is broken.

Instructions

  1. Modify the Code to Use Environment Variables

    • Update both the React and Node.js projects to use environment variables.

      • For the React frontend, modify the API endpoint URL so that it reads from an environment variable. Use a .env file in the project root to store this variable, and ensure it follows the naming convention (VITE_BE_URL for Vite projects).

      • In the Node.js backend, change the MongoDB connection string to read from an environment variable (MONGO_URI).

  2. Pushing the Project to GitHub

    • Initialize a Git repository for both projects if not already done.

    • Commit your changes and push both the React and Node.js projects to separate repositories on GitHub.

  3. Create Render Accounts and Deploy

    • Create a Render Account if you don’t already have one.

    • Deploy the Backend:

      • Create a new Web Service in Render.

      • Connect the Node.js repository from GitHub.

      • Set up the environment variables in Render for the MongoDB URI (MONGO_URI).

      • Deploy the service and note the deployed URL.

    • Deploy the Frontend:

      • Create a new Static Site in Render.

      • Connect the React repository from GitHub.

      • Set up the environment variable (VITE_BE_URL) to point to the backend URL obtained from the previous step.

      • Deploy the static site.

  4. Testing Your Application

    • Once both the frontend and backend are deployed, navigate to your deployed frontend site.

    • Click the ‘Click Me!’ button.

    • If everything is configured correctly and your backend returns a successful response, you should see a smiley face emoji appear.

    • If there is an error in configuration, a frown emoji will be displayed instead.

Notes

  • Ensure that you do not push your .env files to GitHub. Make sure .gitignore includes these files.

  • Environment variables are crucial to maintain security for sensitive information such as API URLs and database URIs.

Helpful Tips

  • For React, use the import.meta.env syntax to access environment variables that start with VITE_.

  • For Node.js, access environment variables using process.env.VARIABLE_NAME.

  • Double-check that the URLs in your environment variables match the deployed URLs on Render.

  • Use Render’s Logs feature to debug any issues during deployment.
